1.패키지-쉽게 폴더(디렉터리)로 생각
-소스코드를 구분하여 저장하기 위한 폴더
-패키지가 다르면 같은이름의 클래스(interface)를 작성 가능
	ex)java.util ->Date 클래스 존재
	   java.sql -> Date 클래스 존재
-패키지 선언법
*******소스 코드 작성시 가장 첫줄에만 작성
*******문법)package 패키지명;
		     └도메인(역방향).사용자업무명
			└이유:개발자가 수주따오면 회사마다 이름이같아 헷갈릴수있다(차별화된 패키지명 필요)

2.이클립스의 컴파일 하는법
	이클립스는 패키지를 사용하여 package compile 필요
문법: javac -d   .    test.java
	   │ └생성된 패키지 안에 바이트 코드를 넣어라
	   └패키지명으로 디렉터리 생성해라

	
********패키지 컴파일된 클래스를 실행*************
주의
1)가장 외부 패키지명이 보이는 위치에서만 실행가능
	└a.b.c ->패키지면 a에서 실행

2)패키지 실행->
	java 패키지명.바이트코드명

2.이클립스 사용법

-장점 1.소스코드작성 2.디버깅 3.배포 ->개발에 필요한 도구들이 있는 무료도구

-eclipse.org 다운로드
-project 로 관리한다

workspae - project - bin -package - 바이트코드 -실행
		└src -package - source code -컴파일

소스코드를 저장하면 컴파일이 자동으로 되고 -bin폴더내에 바이트 코드들이 자동으로 배포된다
run실행만 해주면 된다.
